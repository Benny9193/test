Day 25 — Simultaneous Bidirectional Hybrid Attack Challenge

Goal: Simulate two coordinated red team intrusions — one starting in the cloud and moving into the on-premises environment, the other starting on-premises and pivoting into the cloud — running concurrently to stress-test blue team detection and response under intersecting hybrid attack paths.

Step 1 — Attack Scenario Design (Red Team)

  • Red Team A (Cloud-first): Compromise a cloud account (Azure AD or SaaS credentials), Discover hybrid AD sync or VPN connections, Pivot down to on-prem servers or Domain Controller.
  • Red Team B (On-prem-first): Compromise an on-prem Windows host or service account, Enumerate hybrid AD sync configurations, Pivot upward to access cloud services or consoles.

Step 2 — Environment Preparation (Blue Team)

  • Ensure monitoring visibility across: Cloud logs (Azure Sentinel, AWS GuardDuty, CloudTrail), On-prem logs (Sysmon, Security Event Logs, WEF)
  • Set correlation rules for: Simultaneous unusual logons in both environments, Multiple pivot attempts across hybrid boundaries, Service account and admin account activity anomalies

Step 3 — Concurrent Red Team Execution

  • Red Team A begins cloud compromise and starts pivoting into on-prem.
  • Red Team B begins on-prem compromise and starts pivoting into cloud.
  • Both teams operate simultaneously, timing key actions to overlap and create confusion.
  • Use both stealth and noisy techniques to complicate blue team analysis.

Step 4 — Blue Team Detection & Response

  • Detect indicators from both attack chains:
    • Cloud: unusual sign-ins, admin role creation, API anomalies
    • On-prem: Event ID 4624 anomalies, Sysmon Event ID 3 network connections, new scheduled tasks or registry changes
  • Manage simultaneous incident handling: Prioritize highest-risk detections first, Contain compromised accounts and sessions in both environments, Isolate affected endpoints and terminate cloud sessions

Step 5 — Debrief

  • Red teams: Present attack chains, pivot techniques, and how they leveraged concurrent timing.
  • Blue team: Present detection timeline, incident prioritization process, and any missed events.
  • Jointly identify improvements for handling multi-front hybrid intrusions.

End of Day 25 Success Criteria:

✅Successful simulation of bidirectional hybrid attacks
✅Detection points triggered across both attack paths
✅Documented improvements for managing complex, concurrent incidents

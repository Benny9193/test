Day 17 — Proactive Threat Hunting

Goal: Conduct a proactive blue team hunt for indicators of compromise (IOCs) and malicious activity within lab.local without prior alerts from the red team, simulating a real-world scenario where defenders must identify stealth attacks on their own.

Step 1 — Establish a Threat Hunting Hypothesis

  • Define the scope and focus of the hunt:
    • Example: "An adversary has compromised a domain user account and is moving laterally."
  • Determine the types of activity you will look for:
    • Unusual logon patterns
    • Rare process executions
    • Suspicious PowerShell usage
    • Persistence indicators

Step 2 — Collect and Centralize Data

  • Pull logs from Sysmon, Event Viewer, and WEF.
  • Ensure visibility into: Security event logs, PowerShell operational logs, Sysmon process and network activity
  • If using a SIEM, load relevant log sources and create temporary hunting dashboards.

Step 3 — Search for Anomalies

  • Query for:
    • Event ID 4624 with rare logon types or odd workstations
    • Sysmon Event ID 1 for uncommon process executions
    • Sysmon Event ID 3 for unusual network connections
    • PowerShell logs with Base64-encoded commands
    • WMI events (Sysmon Event ID 21) tied to non-admin accounts

Step 4 — Investigate Leads

  • For each suspicious event:
    • Identify the originating host and account
    • Trace process ancestry
    • Check for related persistence mechanisms (e.g., scheduled tasks, registry keys)
  • Correlate with historical activity to rule out false positives.

Step 5 — Document Findings

  • Maintain a hunt log including: Event IDs, Hostnames and accounts, Timeline of suspicious activity, Determination: benign or malicious
  • Prioritize any confirmed malicious activity for containment.

Step 6 — Contain and Remediate

  • For confirmed malicious findings: Disable compromised accounts, Remove persistence mechanisms, Isolate affected hosts
  • Update detection rules to catch similar activity in the future

End of Day 17 Success Criteria:

✅Threat hunting hypothesis tested and investigated
✅At least one anomaly detected and analyzed
✅Updated detection coverage based on hunt results
✅Documented process for repeating proactive hunts

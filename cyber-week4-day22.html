Day 22 — Advanced Lab Kickoff & Environment Upgrade

Goal: Set up the upgraded lab environment for the next advanced simulation cycle, expanding beyond the Day 1–21 Windows hacking bootcamp by integrating larger enterprise-scale elements, cloud or hybrid infrastructure, and new adversary emulation capabilities.

Step 1 — Lab Expansion Planning

  • Define the scope for the upgraded environment:
    • Number of hosts (Windows, Linux, cloud)
    • Network segmentation (internal, DMZ, cloud connectivity)
    • Active Directory forest/domain complexity
  • Map objectives for the new training cycle:
    • More complex multi-stage intrusions
    • Multi-adversary simultaneous campaigns
    • Cross-platform attack vectors

Step 2 — Infrastructure Upgrades

  • Add new systems:
    • At least one Linux server for cross-platform targeting
    • A cloud-connected Windows Server (Azure or AWS integration)
    • Workstations with varied OS versions for realistic diversity
  • Implement VLANs or virtual networks to simulate enterprise segmentation.

Step 3 — Cloud Integration

  • Deploy a small cloud lab:
    • Azure Active Directory or hybrid AD sync
    • Cloud-hosted Windows and Linux VMs
    • Simulated SaaS applications for attack surfaces
  • Prepare cloud monitoring tools (Azure Sentinel, AWS GuardDuty, etc.).

Step 4 — New Adversary Emulation Playbooks

  • Select additional threat actors from MITRE ATT&CK to target cloud and hybrid environments.
  • Build red team playbooks for: Cloud credential theft, Identity federation attacks, Cloud persistence mechanisms

Step 5 — Detection Enhancements

  • Extend Sysmon and logging policies to new on-prem and cloud systems.
  • Integrate cloud-native logging (Azure Log Analytics, AWS CloudTrail) into central monitoring.
  • Create correlation rules for hybrid attack paths.

Step 6 — Validation & Baseline

  • Perform initial red team tests on new infrastructure.
  • Blue team records baseline activity and validates visibility across all environments.
  • Confirm detection triggers for both on-prem and cloud events.

End of Day 22 Success Criteria:

✅Upgraded lab with on-prem, cloud, and hybrid systems operational
✅Adversary playbooks prepared for advanced scenarios
✅Monitoring and detection extended to all environments
✅Baseline established for the next advanced simulation cycle
